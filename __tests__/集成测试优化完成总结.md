### 1. 扩展Mock数据库 (`__tests__/mocks/responses.ts`)

#### 新增Mock场景

- **用户语气**: 焦虑急躁、过度乐观、犹豫纠结、平静理性
- **专业领域**: 医疗决策、法律决策、大额投资
- **模式对比**: iPhone购买、比特币投资

- 保持与真实API相同的接口和返回格式


### 3. Mock数据质量优化

#### 文本长度控制
- **Fast模式**: <400字符（简短输出）
- **Standard模式**: >400字符（标准输出）
- **Complete模式**: >800字符（详细输出，包含情景模拟和战略路线图）

#### 数据一致性
- Entropy值符合场景要求（信息不足20-30，信息充足70-100）
- Score与dimensions平均值保持一致
- Risk_level与risk维度对应
- 非决策场景正确返回null

#### 多轮对话支持
- 检测历史消息中的关键词
- 根据上下文生成更详细的响应
- Entropy逐步提升（第一轮20-30，第二轮50-80）
- 引用之前的信息（"根据您提供的信息"）

## 使用方式

### Mock模式（推荐）
```bash

npm run test:mock

# 只运行合约测试
npm run test:mock -- __tests__/contract.test.ts

# 只运行多轮对话测试
npm run test:mock -- __tests__/multiturn.test.ts
```

### 真实API模式
```bash
# 设置环境变量
export TEST_API_KEY=your_api_key
```
