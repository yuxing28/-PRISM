# 如何运行自动化测试

## 📋 前置准备

### 1. 安装依赖

确保所有测试依赖已安装：

```bash
cd decision-website
npm install
```

### 2. 配置 API Key（可选）

如果要运行真实API测试，需要配置API Key：

```bash
# 复制环境变量模板
copy .env.test.example .env.test

# 编辑 .env.test 文件，填入你的 API Key
# TEST_API_KEY=your_deepseek_api_key_here
```

**注意**：如果不配置API Key，可以使用Mock模式运行测试。

---

## 🚀 运行测试

### 方式一：Mock模式（推荐，快速）

不需要API Key，使用模拟数据，秒级完成：

```bash
npm run test:mock
```

### 方式二：运行单元测试

只运行单元测试（不调用API）：

```bash
npm run test:unit
```

### 方式三：运行所有测试

运行所有测试（包括需要API的测试）：

```bash
npm test
```

**注意**：这需要配置API Key，且需要较长时间（可能需要几分钟）。

### 方式四：运行特定测试文件

```bash
# 只运行 Store 测试
npm test -- __tests__/unit/store.test.ts

# 只运行 JSON 解析测试
npm test -- __tests__/unit/json-parsing.test.ts

# 只运行用户画像测试
npm test -- __tests__/unit/user-profile.test.ts

# 只运行 JSON 重试测试
npm test -- __tests__/unit/json-retry.test.ts
```

### 方式五：Watch模式（开发时使用）

```bash
npm run test:watch
```

这会监听文件变化，自动重新运行测试。

### 方式六：生成覆盖率报告

```bash
npm run test:coverage
```

---

## 📊 测试分类

### 单元测试（不需要API）

这些测试不需要API Key，运行速度快：

- ✅ `__tests__/unit/json-parsing.test.ts` - JSON解析逻辑
- ✅ `__tests__/unit/store.test.ts` - Store状态管理
- ✅ `__tests__/unit/user-profile.test.ts` - 用户画像学习
- ✅ `__tests__/unit/json-retry.test.ts` - JSON重试机制

运行命令：
```bash
npm run test:unit
```

### 集成测试（需要API或Mock）

这些测试需要API Key或使用Mock模式：

- ⚠️ `__tests__/contract.test.ts` - 合约测试（Prompt输出规范）
- ⚠️ `__tests__/multiturn.test.ts` - 多轮对话测试

运行命令（Mock模式）：
```bash
npm run test:mock
```

运行命令（真实API）：
```bash
# 需要先配置 .env.test
npm test -- __tests__/contract.test.ts
```

## 🔧 常见问题

### Q1: 提示 "Cannot find module '@/lib/store'"

**原因**：TypeScript路径映射配置问题。

**解决**：
1. 检查 `tsconfig.json` 中的 `paths` 配置
2. 检查 `jest.config.js` 中的 `moduleNameMapper` 配置

### Q2: 测试运行很慢

**解决**：使用Mock模式
```bash
npm run test:mock
```

### Q3: API Key 相关错误

**解决方案1**：使用Mock模式（推荐）
```bash
npm run test:mock
```

**解决方案2**：配置API Key
```bash
# 创建 .env.test 文件
echo TEST_API_KEY=your_key_here > .env.test
```

### Q4: 某些测试失败

**调试步骤**：
1. 运行单个测试文件查看详细错误
   ```bash
   npm test -- __tests__/unit/store.test.ts --verbose
   ```

2. 检查错误信息
3. 如果是API相关错误，尝试Mock模式

### Q5: 提示找不到测试文件

**检查**：
- 测试文件是否在 `__tests__` 目录下
- 文件名是否以 `.test.ts` 或 `.test.tsx` 结尾
- `jest.config.js` 中的 `testMatch` 配置是否正确

## 📝 测试脚本说明

在 `package.json` 中已配置的测试脚本：

```json
{
  "scripts": {
    "test": "jest",                          // 运行所有测试
    "test:watch": "jest --watch",            // Watch模式
    "test:coverage": "jest --coverage",      // 生成覆盖率
    "test:unit": "jest __tests__/unit",      // 只运行单元测试
    "test:contract": "jest __tests__/contract.test.ts",  // 合约测试
    "test:multiturn": "jest __tests__/multiturn.test.ts", // 多轮对话测试
    "test:mock": "cross-env MOCK_MODE=true jest"  // Mock模式
  }
}
```

## 💡 最佳实践

1. **开发时**：使用 `npm run test:watch` 实时查看测试结果
2. **快速验证**：使用 `npm run test:unit` 或 `npm run test:mock`
3. **发布前**：运行 `npm test` 进行完整测试
4. **CI/CD**：使用 `npm run test:mock` 或配置API Key后运行完整测试
