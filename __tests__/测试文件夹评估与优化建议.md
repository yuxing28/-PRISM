# 测试文件夹评估与优化建议

## 📊 当前测试结构评估

### ✅ 优点

1. **测试分类清晰**
   - 合约测试 (contract.test.ts) - 验证Prompt输出规范
   - 单元测试 (unit/json-parsing.test.ts) - 验证解析逻辑
   - 多轮对话测试 (multiturn.test.ts) - 验证上下文管理
   - 测试用例库 (test-cases/scenarios.json) - 结构化测试数据

2. **测试覆盖全面**
   - 50+ 测试场景
   - 覆盖信息不足、信息充足、边界情况等
   - 包含四种模式的差异测试

3. **工具函数完善**
   - test-helpers.ts 提供了完整的辅助函数
   - 支持Mock模式和真实API模式
   - JSON解析和验证逻辑独立

4. **文档详细**
   - README.md 提供了完整的使用指南
   - 包含快速开始、测试模式、常见问题等

### ❌ 问题与不足

#### 1. **配置问题**

**问题**：
- `jest.config.js` 文件被截断，配置不完整
- `package.json` 中缺少一些必要的测试脚本
- API Key 硬编码在 setup.ts 中（安全风险）

**影响**：
- 测试可能无法正常运行
- 配置不完整导致路径解析失败

#### 2. **测试依赖问题**

**问题**：
- 测试依赖本地服务器运行 (localhost:3000)
- 没有集成测试环境的自动启动
- 缺少 CI/CD 配置

**影响**：
- 测试运行前需要手动启动服务器
- 不适合自动化测试流程

#### 3. **测试覆盖不足**

**缺失的测试**：
- ❌ 前端组件测试（React组件）
- ❌ Store状态管理测试（Zustand）
- ❌ 用户画像学习逻辑测试
- ❌ JSON重试机制测试
- ❌ 性能测试
- ❌ 端到端测试（E2E）

#### 4. **测试数据管理**

**问题**：
- scenarios.json 数据量较大但缺少分类管理
- 没有测试数据生成工具
- 缺少测试结果的持久化和对比

#### 5. **Mock数据不够真实**

**问题**：
- getMockResponse 函数过于简单
- 没有模拟不同的边界情况
- 缺少错误场景的Mock

#### 3.1 添加测试服务器自动启动

创建 `__tests__/utils/test-server.ts`：
```typescript
/**
 * 测试服务器管理
 */
import { spawn, ChildProcess } from 'child_process';

let serverProcess: ChildProcess | null = null;

export async function startTestServer(): Promise<void> {
  return new Promise((resolve, reject) => {
    serverProcess = spawn('npm', ['run', 'dev'], {
      stdio: 'pipe',
      shell: true,
    });

    serverProcess.stdout?.on('data', (data) => {
      const output = data.toString();
      if (output.includes('Ready')) {
        resolve();
      }
    });

    serverProcess.on('error', reject);

    // 超时保护
    setTimeout(() => reject(new Error('服务器启动超时')), 30000);
  });
}

export function stopTestServer(): void {
  if (serverProcess) {
    serverProcess.kill();
    serverProcess = null;
  }
}
```
