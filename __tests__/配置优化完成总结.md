## 主要改动

### 1. 修复 `jest.config.js` 配置

#### 问题
- 使用了deprecated的 `globals` 配置
- `isolatedModules` 应该在 `transform` 配置中设置

#### 解决方案
```javascript
// ❌ 旧配置（deprecated）
globals: {
  'ts-jest': {
    isolatedModules: true,
  },
}

// ✅ 新配置（推荐）
transform: {
  '^.+\\.tsx?$': ['ts-jest', {
    tsconfig: {
      jsx: 'react',
      esModuleInterop: true,
      allowSyntheticDefaultImports: true,
      isolatedModules: true,  // 移到这里
    },
  }],
}
```

### 2. 更新 `tsconfig.json` 配置

#### 问题
- `isolatedModules` 配置应该在 `tsconfig.json` 中明确声明
- `jsx` 配置应该使用 Next.js 推荐的 `preserve` 模式

#### 解决方案
```json
{
  "compilerOptions": {
    "isolatedModules": true,  // 已存在，保持不变
    "jsx": "preserve",        // 从 "react-jsx" 改为 "preserve"
    // ... 其他配置
  }
}
```
