## 四、核心功能模块

### 4.1 决策分析系统

**三种分析模式**:
1. **快速模式** (5-10分钟): 适用于低价值决策，2个核心维度验证
2. **标准模式** (1-2小时): 中等价值决策，5步验证流程
3. **完整模式** (3-5小时): 战略级决策，7步深度验证

**红队对抗模式**: 扮演"魔鬼代言人"，专门寻找决策漏洞

### 4.2 多维度评估体系

六个评估维度 (0-10分):
- **逻辑严密度** (logic): 因果关系是否成立
- **可行性** (feasibility): 执行条件是否具备
- **风险控制** (risk): 最坏情况能否承受
- **价值匹配度** (value): 是否符合用户真实目标
- **时机适当性** (timing): 当前是否适合行动
- **资源充足度** (resource): 资金/时间/精力是否充裕

### 4.3 风险等级划分
- 低风险 / 中等风险 / 高风险 / 致命风险

### 4.4 信息完备度 (Entropy)
- 0-30%: 信息严重不足
- 31-50%: 信息基本够用
- 51-70%: 信息较为完整
- 71-100%: 信息非常充分

## 六、状态管理设计

### 6.1 Store 结构 (Zustand)

```typescript
interface DecisionStore {
  // 多会话管理
  sessions: Session[];
  currentSessionId: string | null;
  
  // 全局状态
  isLoading: boolean;
  currentInput: string;
  apiKey: string;
  
  // UI 状态
  sidebarCollapsed: boolean;
  mobileMenuOpen: boolean;
  mobilePanelOpen: boolean;
  showSettings: boolean;
}

interface Session {
  id: string;
  title: string;
  messages: Message[];
  analysis: AnalysisState;  // 评分数据
  entropyProgress: number;
  isDebateMode: boolean;
  decisionMode: 'fast' | 'standard' | 'complete';
  createdAt: number;
}
```

### 6.2 数据持久化
- 使用 `zustand/middleware/persist`
- 存储到 localStorage
- 支持版本迁移 (v0 → v1)

## 七、API 设计

### 7.1 聊天接口

**端点**: `POST /api/chat`

**请求体**:
```json
{
  "messages": [{ "role": "user", "content": "..." }],
  "isDebateMode": false,
  "decisionMode": "standard",
  "apiKey": "sk-..."
}
```

**响应**: SSE 流式响应

### 7.2 System Prompt 设计

每个模式都包含:
1. **通用模块**: 意图识别、情绪安抚、安全边界、骗局检测
2. **模式专属**: 验证流程、输出风格、评分标准
3. **JSON 输出指令**: 强制要求输出结构化评估数据

### 7.3 JSON 数据解析

前端实现了健壮的 JSON 解析机制:
- 支持 `___JSON_BLOCK_START___` 分隔符
- 支持 ```json 代码块
- 自动修复常见格式问题
- 失败时自动重试 (最多2次)
- 数据验证与默认值填充
