# 智镜 (Prism AI) v2.0 极致优化方案调研报告

## 第一部分：技术架构极致优化 (Technical Optimization)

本部分聚焦于提升 AI 响应延迟、数据隐私及边缘侧处理能力。

| 维度 | 技术原理 | 推荐方案 / 库 | 实施难度 |
| :--- | :--- | :--- | :--- |
| **AI Streaming (RSC)** | 利用 React Server Components (RSC) 将 UI 渲染逻辑移至服务端，通过 `streamUI` 直接返回组件流，避免前端解析 JSON 的性能损耗。 | **Vercel AI SDK (rsc)** / `useChat` + `streamUI` | 4 |
| **推测性解码 (Speculative)** | 在流式输出 JSON 时，前端利用 `Partial JSON Parser` 预判数据结构，在数据未完全到达前就开始渲染占位符或初步 UI。 | **JSON-UI Streaming Patterns** / `partial-json-parser` | 3 |
| **WebLLM & 隐私** | 利用 WebGPU 在浏览器端直接运行轻量级模型（如 Qwen2.5-0.5B），用于意图识别、数据打敏，完全不经过云端。 | **mlc-ai/web-llm** / **Transformers.js v3** | 5 |
| **边缘计算 (Edge)** | 将 Prompt 注入、权限验证移至 Vercel Edge Runtime，利用边缘侧 KV 缓存加速高频请求。 | **Next.js Edge Runtime** + **Vercel KV** | 2 |
| **全方位监控** | 基于代理 (Proxy) 模式拦截 API 请求，无需在业务代码中埋点，即可追踪 Token 消耗、延迟及成本。 | **Helicone** / **Langfuse** | 1 |

### 关键伪代码：Generative UI Streaming
```tsx
// app/api/chat/route.ts
import { streamUI } from 'ai/rsc';
import { deepseek } from '@/lib/ai-provider'; // 封装的 Provider

export async function POST(req: Request) {
  const { messages } = await req.json();

  const result = await streamUI({
    model: deepseek('deepseek-chat'),
    messages,
    text: ({ content }) => <div>{content}</div>,
    tools: {
      showAnalysis: {
        description: '显示多维决策分析仪表盘',
        parameters: z.object({ score: z.number(), risk: z.string() }),
        generate: async ({ score, risk }) => <AnalysisCard score={score} risk={risk} />
      }
    }
  });

  return result.value;
}
```

## 第二部分：UI/UX 设计巅峰进化 (Aesthetics & Interaction)

借鉴 Linear, Raycast 及 Reflect App 的极致细节，为“智镜”量身定制的设计方案。

### 1. 高级材质：WebGL 玻璃折射 (Advanced Glass)
*   **视觉参考**：Apple 官网的动态毛玻璃效果，鼠标掠过时背景有轻微的流体扭曲感。
*   **技术实现**：使用 `react-three-fiber` 配合 `MeshTransmissionMaterial`。在 CSS 玻璃层下方覆盖一层透明的 WebGL Canvas。
*   **心理学依据**：**“触觉反馈错觉”**。通过视觉上的深度（Depth）和折射（Refraction），让非实体界面产生“物理存在感”，提升产品的溢价感。

### 2. 微交互：有机节奏感 (Organic Rhythms)
*   **交互场景优化**：
    *   **点击按钮**：`scale: 0.96` + `spring` 物理反馈（而不是线性动画）。
    *   **AI 正在思考**：不显示 Loading 圈，而是让背景的 3个 Blob 光斑频率同步到 0.5Hz 的**低频呼吸**，并伴随细微的色散（Chromatic Aberration）闪烁。
    *   **生成报告**：采用“从底部像素级逐行扩散”的扫光动效。
*   **推荐库**：`framer-motion` (Layout transitions).

### 3. 数据可视化：全息质感 (Holographic Data)
*   **视觉参考**：Reflect App 的图表风格。
*   **技术实现**：
    *   使用 **Nivo (Line/Radar)**，但将线条颜色设置为补色的渐变，并开启 `filter: drop-shadow(0 0 8px #cyan)`。
    *   在雷达图底部叠加一层 `radial-gradient` 的辉光。
*   **Why**：**“数据即光芒”**。将冰冷的数据拟态为发光的能量源，符合“推演引擎”的概念。

### 4. 极致排版：等宽美学 (Typography)
*   **字体推荐**：
    *   **数据展示**：`JetBrains Mono`（权重 500）。其大字高（x-height）和独特的连字（Ligatures）在暗色玻璃背景下极易阅读。
    *   **系统正文**：`Geist Sans` (Vercel 官方字体)。极简的几何结构，符合科技严肃感。
*   **排版逻辑**：
    *   行高（line-height）统一设置为 `1.5 - 1.6`。
    *   字间距（letter-spacing）在数据密集区适当增加 `0.02em` 以防视觉粘连。
