## 🎯 优化方案（按优先级）

### 优先级1：Prompt可控协议化（立即实施）

#### 1.1 消除Prompt内部冲突

**目标**：解决"不做分析"与"给评估值"的矛盾，引入明确的"占位值"概念。

**当前问题**（route.ts 实际代码）：
- 第264行：要求"不要进行完整的多维度分析"
- 第265行：要求"其他维度给出初步估计值"
- 第191行：要求"即使信息不足，也要基于已有信息给出评估值"

**矛盾后果**（chat-area.tsx第61-63行可见）：
```typescript
// 前端不得不修复模型返回的score=0
if (score === 0) {
    score = Math.max(1, Math.min(10, Math.round(avgScore)));
}
```

**正确方案**：修改 `route.ts` 第265行和第319行

```typescript
// ========== 标准模式（第265行）==========
// 修改前（有矛盾）
6. 【重要】此时的JSON数据：entropy设为20-30（表示信息不足），其他维度给出初步估计值

// 修改后（明确占位值）
6. 【重要】此时的JSON数据（占位值模式）：
   {
     "score": 5,  // 固定占位值，不代表实际评分
     "risk_level": "信息不足，待评估",
     "entropy": 25,  // 20-30之间
     "risk_factors": ["需要补充更多决策背景"],
     "dimensions": {
       "logic": 5,      // 所有维度统一5分（占位）
       "feasibility": 5,
       "risk": 5,
       "value": 5,
       "timing": 5,
       "resource": 5
     }
   }
   
   【关键】在正文末尾添加：
   "*上述评分为占位值，补充信息后将进行正式分析*"

// ========== 完整模式（第319行）同样修改 ==========
6. 【重要】此时的JSON数据（占位值模式）：
   {
     "score": 5,
     "risk_level": "信息严重不足，待评估",
     "entropy": 20,  // 15-25之间
     "risk_factors": ["需要补充详细决策信息"],
     "dimensions": { /* 所有维度5分 */ }

   在正文末尾添加："*上述评分为占位值，补充信息后将进行战略级分析*"
```

**为什么这样改？**

1. **消除Prompt矛盾**：不再要求"既不分析又评估"，而是明确"给占位值"
2. **前端可识别**：

   ```typescript
   const isPlaceholder = analysis.entropy < 30 && analysis.score === 5;
   ```

3. **用户体验**：明确告知"这是占位，不是真实评分"，避免误解
4. **修复前端bug**：chat-area.tsx第61-63行的修复逻辑自动失效（score不再是0）

#### 1.2 强化JSON一致性规则

**方案**：在 `JSON_INSTRUCTION` 中添加一致性约束

```typescript
// route.ts 第159行后添加
===========================================
⚠️ 【JSON一致性约束 - 强制执行】
===========================================

1. score 与 dimensions 的关系：
   - score 必须接近六个维度的平均值（±1分误差内）
   - 公式：score ≈ (logic + feasibility + risk + value + timing + resource) / 6

2. risk_level 与 risk 维度的对应：
   - risk 维度 8-10分 → risk_level = "低风险"
   - risk 维度 5-7分 → risk_level = "中等风险"
   - risk 维度 2-4分 → risk_level = "高风险"
   - risk 维度 0-1分 → risk_level = "致命风险"

3. 禁止0分（除非信息不足）：
   - 任何维度评分不要给0分
   - 如果确实无法评估，请给5分（中等）
   - 唯一例外：entropy < 30 时，可以所有维度设为0表示"未评估"

4. entropy 与分析深度的对应：
   - entropy < 30: 信息严重不足，只能收集问题
   - entropy 30-50: 信息基本够用，但有关键缺失
   - entropy 51-70: 信息较完整，可信度中等
   - entropy 71-100: 信息非常充分，高可信度
```

#### 1.3 稳定可读结构

**方案**：修改 `COMMON_PREFIX` 的"回复结构"部分

```typescript
// route.ts 第143-156行，替换为：
【回复结构 - 强制执行】

当且仅当进入决策相关分析时（不包括问候/闲聊/跑题），你的正文必须满足：

1. 开头一句明确结论（三选一）：
   - "我建议做" / "我不建议做" / "需要更多信息才能给出建议"
   - 【禁止模棱两可】不要说"可以考虑"、"看情况"

2. 主体分析：
   - 第一段：说清楚这是什么决策，核心问题在哪
   - 第二段：最关键的2-3个理由（支持或反对）
   - 第三段：最值得注意的1-2个风险点

3. 结尾行动建议：
   - 如果建议做：下一步具体行动（验证什么/准备什么）
   - 如果不建议做：替代方案或止损建议
   - 如果信息不足：明确列出缺失的关键信息（2-4项）

【关键】：
- 每段之间空一行
- 用自然段落，不要用 Markdown 列表符号
- 第一段信息不足时，允许用"- "列举问题
```

```typescript
// 模拟API调用
async function testModeOutput(
  mode: 'fast' | 'standard' | 'complete', 
  userInput: string
) {
  const response = await fetch('/api/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      messages: [{ role: 'user', content: userInput }],
      isDebateMode: false,
      decisionMode: mode,
      apiKey: process.env.TEST_API_KEY,
      userMemory: { /* 默认值 */ }
    })
  });
  
  const reader = response.body!.getReader();
  const decoder = new TextDecoder();
  let fullText = '';
  
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    fullText += decoder.decode(value);
  }
  
  return fullText;
}
```

```typescript
export function shouldCompressContext(messages: Message[]): boolean {
  const totalTokens = estimateTokens(messages);
  const TOKEN_THRESHOLD = 6000; // DeepSeek-V3支持64K，但保留安全边界
  return totalTokens > TOKEN_THRESHOLD;
}

/**
 * 简单token估算（1 token ≈ 1.5个中文字符）
 */
function estimateTokens(messages: Message[]): number {
  const totalChars = messages.reduce((sum, msg) => sum + msg.content.length, 0);
  return Math.ceil(totalChars / 1.5);
}

${summary.openQuestions.length > 0 ? `待解决问题：\n${summary.openQuestions.map(q => `- ${q}`).join('\n')}` : ''}
---
`
  };
  
  return [summaryMessage, ...recentMessages];
}
```

```typescript
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';

// 简单的内存限流器
const requestCounts = new Map<string, { count: number; resetTime: number }>();

export function middleware(request: NextRequest) {
    // 只对API路由生效
    if (!request.nextUrl.pathname.startsWith('/api/chat')) {
        return NextResponse.next();
    }
    
    // 获取客户端IP
    const ip = request.ip || request.headers.get('x-forwarded-for') || 'unknown';
    
    const now = Date.now();
    const windowMs = 60 * 1000; // 1分钟窗口
    const maxRequests = 30; // 每分钟最多30次请求（平均2秒一次）
    
    let record = requestCounts.get(ip);
    
    // 重置窗口
    if (!record || now > record.resetTime) {
        record = { count: 0, resetTime: now + windowMs };
        requestCounts.set(ip, record);
    }
    
    // 检查限流
    if (record.count >= maxRequests) {
        return NextResponse.json(
            { error: '请求过于频繁，请稍后再试' },
            { status: 429 }
        );
    }
    
    record.count++;
    
    return NextResponse.next();
}

export const config = {
    matcher: '/api/chat'
};
```
